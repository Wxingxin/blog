# 一、HTTP 基础（必问）

## 1️⃣ HTTP 是什么？特点？

**答：**

* 超文本传输协议
* **无状态**
* 基于请求–响应模型
* 通常运行在 TCP 之上

---

## 2️⃣ HTTP 请求由哪些部分组成？

**答：**

* 请求行（方法 / URL / 协议版本）
* 请求头（Headers）
* 请求体（Body，GET 通常没有）

---

## 3️⃣ 常见 HTTP 方法及区别？

| 方法      | 作用   |
| ------- | ---- |
| GET     | 查询资源 |
| POST    | 提交数据 |
| PUT     | 整体更新 |
| PATCH   | 局部更新 |
| DELETE  | 删除   |
| OPTIONS | 预检请求 |

---

## 4️⃣ GET 和 POST 的区别？

**答：**

* GET 用于查询，POST 用于提交
* GET 参数在 URL，POST 在 Body
* GET 可缓存，POST 默认不可
* GET 有长度限制（浏览器）

---

# 二、状态码（⭐高频）

## 5️⃣ 常见 HTTP 状态码？

| 分类  | 示例                         |
| --- | -------------------------- |
| 2xx | 200 成功                     |
| 3xx | 301 重定向                    |
| 4xx | 401 未认证 / 403 禁止 / 404 未找到 |
| 5xx | 500 服务器错误 / 502 网关错误       |

---

## 6️⃣ 301 和 302 区别？

**答：**

* 301：永久重定向（利于 SEO）
* 302：临时重定向

---

# 三、HTTPS & 安全（必问）

## 7️⃣ HTTPS 做了什么？

**答：**

* 加密数据（防窃听）
* 校验身份（防伪装）
* 保证完整性（防篡改）

---

## 8️⃣ HTTPS 的加密过程？

**答：**

1. 客户端请求证书
2. 服务器返回公钥证书
3. 客户端生成对称密钥
4. 用公钥加密传给服务器
5. 后续通信用对称加密

👉 **非对称 + 对称结合**

---

## 9️⃣ HTTP 和 HTTPS 区别？

| HTTP  | HTTPS  |
| ----- | ------ |
| 明文    | 加密     |
| 80 端口 | 443 端口 |
| 不安全   | 安全     |
| 无证书   | 有证书    |

---

# 四、TCP / 网络模型（⭐中高级）

## 🔟 TCP 三次握手？

**答：**

1. 客户端：SYN
2. 服务端：SYN + ACK
3. 客户端：ACK

👉 建立可靠连接

---

## 1️⃣1️⃣ 为什么需要三次？

**答：**

* 防止历史连接请求
* 双方确认收发能力

---

## 1️⃣2️⃣ TCP 四次挥手？

**答：**

1. 客户端 FIN
2. 服务端 ACK
3. 服务端 FIN
4. 客户端 ACK

---

## 1️⃣3️⃣ HTTP 在哪一层？

**答：**

* **应用层**
* TCP：传输层
* IP：网络层

---

# 五、缓存机制（⭐超高频）

## 1️⃣4️⃣ 强缓存和协商缓存？

**答：**

* 强缓存：

  * Cache-Control
  * Expires
* 协商缓存：

  * ETag
  * Last-Modified

---

## 1️⃣5️⃣ Cache-Control 常见值？

```text
max-age
no-cache
no-store
public
private
```

---

## 1️⃣6️⃣ ETag 和 Last-Modified 区别？

**答：**

* ETag 更精确
* Last-Modified 有秒级限制

---

# 六、跨域（必问）

## 1️⃣7️⃣ 什么是跨域？

**答：**

* 协议 / 域名 / 端口不同

---

## 1️⃣8️⃣ 解决跨域方式？

**答：**

* CORS（最常用）
* JSONP（已淘汰）
* 反向代理
* postMessage

---

## 1️⃣9️⃣ CORS 简单请求 vs 复杂请求？

**答：**

* 简单请求：直接发
* 复杂请求：先发 **OPTIONS 预检**

---

# 七、性能优化（加分）

## 2️⃣0️⃣ 减少网络请求的方法？

**答：**

* 合理缓存
* CDN
* HTTP/2
* 合并请求
* 懒加载

---

## 2️⃣1️⃣ HTTP/1.1 vs HTTP/2？

| HTTP/1.1 | HTTP/2 |
| -------- | ------ |
| 串行       | 多路复用   |
| 文本头      | 二进制    |
| 队头阻塞     | 无      |

---

## 2️⃣2️⃣ 什么是 CDN？

**答：**

* 内容分发网络
* 就近访问
* 降低延迟
